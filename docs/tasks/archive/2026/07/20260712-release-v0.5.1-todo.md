# Release v0.5.1

Close out the v0.5.1 release covering the `v0.5.0..HEAD` window (16
commits). Theme: **Sheets data validation + Slides parity + PDF Phase 2**
— additive features and fixes layered on the existing document types, with
no new document type and no new infrastructure.

**Patch (not minor) bump** — every change is backward-compatible and
extends an existing document type or subsystem: no new document type, no
new env/infra (PDF Phase 2 reuses the `pdf-<id>` Yorkie key + blob store
already shipped in v0.5.0; data validation / charts / gradient extend the
existing Sheets & Slides engines). Per the v0.5.0 convention, the minor
bump was reserved for the new PDF document type + blob-store infra; neither
recurs here, so v0.5.1 is a patch.

## Highlights

- **Sheets — data validation**: in-cell checkboxes (#460), list/dropdown
  control + a Data validation side panel (#464). Formula correctness:
  `#NUM!` on numeric overflow instead of `Infinity` (#230), boolean
  coercion in `SUMPRODUCT` (#199).
- **Sheets — xlsx import**: cell styles, column widths, and date serials on
  `.xlsx` import (#467).
- **Slides**: PPTX chart import & native render (#459); layout background +
  geometry inheritance across masters (#462); title-alignment inheritance +
  gradient shape fills on PPTX import (#463); linear gradient fill editing
  with a stops-bar (#466).
- **PDF**: Phase 2 — share links, page-anchored comments, and presence
  (#456); document rename in the viewer (#455); hardened pdfjs-assets dev
  middleware (#457).
- **Documents list**: stable last-modified ordering via a Yorkie
  `DocumentRootChanged` webhook feeding Postgres `updatedAt` (#465).
- **Workspaces**: redirect already-members to the workspace on invite
  accept (#461).
