# Release v0.6.1

Close out the v0.6.1 release covering the `v0.6.0..HEAD` window (7
commits). Theme: **Share Link view analytics + multi-file upload +
unified document permissions + Notes CLI/backend plumbing**.

**Patch (not minor) bump** — this window ships features and polish on the
existing document types with **no new document type, no new package, and
no new infra** in this repo (the analytics warehouse is reused Yorkie
Kafka/StarRocks stack and degrades to a no-op when unconfigured). Per the
minor-reserved-for-new-doc-type-+-infra convention, this is a patch.

## Highlights

- **Share Link view analytics** (#491): reuse Yorkie's Kafka + StarRocks
  OLAP stack (`analytics` ns) — NestJS kafkajs producer + Routine Load
  into `wafflebase.view_events`, mysql2 reader, per-document manager
  dashboard (views / unique / returning / dwell + per-link breakdown) and
  a workspace-aggregate view, `sendBeacon` frontend hook; degrades to a
  no-op without Kafka/StarRocks. DevOps infra lives in the separate
  `yorkie-team/devops` repo.
- **Documents multi-file upload** (#492): Google-Drive-style
  drag-and-drop + multi-select on the documents list — hand-rolled upload
  queue (module-singleton, no state library) routing each file to its
  matching document type (xlsx/docx/pptx/pdf), unsupported skipped, fixed
  bottom-right upload panel with per-file progress/retry.
- **Unified document permissions** (#484): share links + delete/move
  gated by a single manager tier (workspace owner or document author).
- **Notes plumbing** (#483): `wafflebase` CLI notes namespace + backend
  note content endpoint.
- **Docs site** (#485): documentation site refreshed for v0.6.0 — Notes,
  PDF, data validation.
- **Polish** (#493): type filter buttons use a `ToolTip`.
